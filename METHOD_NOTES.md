# 方法笔记

## 算法总体流程

```
输入字形图像
    ↓
图像预处理（灰度化、二值化、去噪）
    ↓
骨架提取（Zhang-Suen / 形态学细化）
    ↓
骨架点排序 → 初始轨迹点序列
    ↓
强化学习局部优化（DDPG）
    ↓
优化后的机器人末端轨迹
```

## 1. 图像预处理

- 灰度化：将RGB图转为灰度图
- 二值化：Otsu自适应阈值
- 去噪：中值滤波 / 高斯滤波

## 2. 骨架提取

- 算法：skimage.morphology.thin（形态学细化，优于手工 Zhang-Suen）
- 实验对比：thin (1.7% 交叉点) >> skeletonize (3.0%) >> Zhang-Suen (29.3%)
- 输出：单像素宽度的字形骨架
- 后处理：prune_skeleton（移除 <10px 的毛刺分支）

## 3. 笔画提取（简化图 + 端点配对）

核心思路：
1. 建全图 G(V,E)，V=所有骨架像素，E=8-邻接
2. 找端点 (deg=1) 和交叉点 (deg≥3)，交叉点 8-连通聚类为"交叉分量"
3. 所有桥接连通的交叉分量合并为"超级交叉区"
4. 收缩 deg-2 链点 → 简化图（节点=端点+交叉分量，超边=链路径）
5. 在每超级交叉区按方向连续性贪心配对，剩余强制造配对
6. 排序（自上而下、自左而右）+ 定向（去程从起笔到收笔）

## 4. B样条平滑

- 逐笔画独立平滑（非全轨迹拼接），按笔画长度比例分配采样点
- splprep + splev，s=2.0 控制平滑强度
- 笔画间用 nan,nan 行分隔（表示机器人抬笔动作）

## 5. 强化学习优化（DDPG）

### 问题建模
- 将轨迹优化建模为 MDP：每个轨迹点是一个决策步
- 实验设计：先对平滑轨迹加高斯噪声（σ=6px）模拟示教误差，再让 DDPG 恢复

### 状态空间 (57 维)
- 归一化位置 (y/H, x/W)：2 维
- 局部二值图 patch (7×7=49)：捕捉当前点周围的字形信息
- 到最近前景像素的距离 (归一化)：1 维
- 前后点方向向量 (dy,dx)×2：4 维
- 局部曲率 (1-cosθ)/2：1 维

### 动作空间
- 连续位移 (Δy, Δx) ∈ [-3, 3] 像素

### 奖励函数
- R = 0.3·R_foreground + 0.3·R_inside + 0.3·R_smooth + 0.1·R_boundary
- R_foreground：向最近前景靠近的改进量
- R_inside：在前景区域内为正奖赏
- R_smooth：相邻点距离 ≤8px，超出扣分
- R_boundary：出图像边界扣 -0.5

### 网络结构
- Actor：state(57) → FC(128)→ReLU→FC(128)→ReLU→FC(2)→Tanh×max_action
- Critic：state→FC(128) + action→FC(128) → cat(256)→FC(128)→ReLU→FC(1)
- DDPG：off-policy，经验回放 (10000)，软更新 (τ=0.005)，OU 噪声探索
- 学习率：actor 1e-4, critic 1e-3, γ=0.95

### 初步结果
- 永字 5 笔画，200 轮/笔画，总体 Chamfer 降低约 25%
- 改进空间主要在笔画交叉区域（当前 74% 骨架覆盖率）

## 备选方案（如RL训练不稳定）

- 使用"强化学习思想 + 局部搜索/策略迭代"
- 使用改进Q-learning替代DDPG
- 使用PPO替代DDPG
