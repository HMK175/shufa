# 方法笔记

## 算法总体流程

```
输入字形图像
    ↓
图像预处理（灰度化、二值化、去噪）
    ↓
骨架提取（Zhang-Suen / 形态学细化）
    ↓
骨架点排序 → 初始轨迹点序列
    ↓
强化学习局部优化（DDPG）
    ↓
优化后的机器人末端轨迹
```

## 1. 图像预处理

- 灰度化：将RGB图转为灰度图
- 二值化：Otsu自适应阈值
- 去噪：中值滤波 / 高斯滤波

## 2. 骨架提取

- 算法：skimage.morphology.thin（形态学细化，优于手工 Zhang-Suen）
- 实验对比：thin (1.7% 交叉点) >> skeletonize (3.0%) >> Zhang-Suen (29.3%)
- 输出：单像素宽度的字形骨架
- 后处理：prune_skeleton（移除 <10px 的毛刺分支）

## 3. 笔画提取（简化图 + 端点配对）

核心思路：
1. 建全图 G(V,E)，V=所有骨架像素，E=8-邻接
2. 找端点 (deg=1) 和交叉点 (deg≥3)，交叉点 8-连通聚类为"交叉分量"
3. 所有桥接连通的交叉分量合并为"超级交叉区"
4. 收缩 deg-2 链点 → 简化图（节点=端点+交叉分量，超边=链路径）
5. 在每超级交叉区按方向连续性贪心配对，剩余强制造配对
6. 排序（自上而下、自左而右）+ 定向（去程从起笔到收笔）

## 4. B样条平滑

- 逐笔画独立平滑（非全轨迹拼接），按笔画长度比例分配采样点
- splprep + splev，s=2.0 控制平滑强度
- 笔画间用 nan,nan 行分隔（表示机器人抬笔动作）

## 5. 强化学习优化（DDPG）

### 问题建模
- 将轨迹优化建模为 MDP：每个轨迹点是一个决策步
- 实验设计：先对平滑轨迹加高斯噪声（σ=6px）模拟示教误差，再让 DDPG 恢复

### 状态空间 (57 维)
- 归一化位置 (y/H, x/W)：2 维
- 局部二值图 patch (7×7=49)：捕捉当前点周围的字形信息
- 到最近前景像素的距离 (归一化)：1 维
- 前后点方向向量 (dy,dx)×2：4 维
- 局部曲率 (1-cosθ)/2：1 维

### 动作空间
- 连续位移 (Δy, Δx) ∈ [-3, 3] 像素

### 奖励函数
- R = 0.3·R_foreground + 0.3·R_inside + 0.3·R_smooth + 0.1·R_boundary
- R_foreground：向最近前景靠近的改进量
- R_inside：在前景区域内为正奖赏
- R_smooth：相邻点距离 ≤8px，超出扣分
- R_boundary：出图像边界扣 -0.5

### 网络结构
- Actor：state(57) → FC(128)→ReLU→FC(128)→ReLU→FC(2)→Tanh×max_action
- Critic：state→FC(128) + action→FC(128) → cat(256)→FC(128)→ReLU→FC(1)
- DDPG：off-policy，经验回放 (10000)，软更新 (τ=0.005)，OU 噪声探索
- 学习率：actor 1e-4, critic 1e-3, γ=0.95

### 初步结果
- 永字 5 笔画，200 轮/笔画，总体 Chamfer 降低约 25%
- 改进空间主要在笔画交叉区域（当前 74% 骨架覆盖率）

## 论文方向调整与创新点定位（2026-05-15）

### 1. 方法主线建议

当前实验表明，强化学习模块容易受到初始轨迹质量影响；当骨架拓扑解析存在误分割、误连接或回绕路径时，RL 只能在错误轨迹基础上做局部微调，难以从根本上修正笔画结构。因此，论文主线建议从“骨架提取 + RL优化”调整为：

```
字形图像输入
    -> 图像预处理
    -> 骨架提取
    -> 骨架拓扑图构建
    -> 笔画路径分割与异常修正
    -> 轨迹平滑与弧长重采样
    -> 机器人末端轨迹输出
```

强化学习模块可作为可选探索或后续工作，不建议作为当前论文的核心方法。

### 2. 推荐论文题目方向

可将题目调整为：

**基于骨架拓扑约束的书法机器人笔画轨迹生成方法**

或：

**结合骨架拓扑分析与笔画先验的书法机器人轨迹生成方法**

该方向更贴合当前工程实现，也更容易通过稳定实验结果支撑。

### 3. 核心创新点建议

不宜只写“引入汉字笔画先验”，因为单纯使用笔画数、部件结构等知识较容易想到，创新性偏弱。建议将创新点具体化为：

**骨架拓扑异常感知的笔画轨迹修正方法**

其核心思想是：将骨架像素转换为图结构，把端点、交叉点、分支边和连通区域作为拓扑对象，然后利用多种可量化约束检测并修正异常轨迹。

可设计的拓扑异常指标包括：

- 回绕率：轨迹路径长度 / 首尾欧氏距离。若该值过大，说明轨迹可能出现闭环或绕行。
- 方向连续性：交叉点前后分支方向夹角。夹角越接近同向或自然转折，越可能属于同一笔。
- 端点可信度：判断端点是真实笔画端点，还是由毛刺、粗笔画边缘或骨架断裂产生的伪端点。
- 笔画数约束：与目标汉字的预期笔画数进行弱约束匹配。
- 部件边界约束：对于左右结构或多部件汉字，限制跨部件误连接。

这样论文创新点不是“查表式笔画先验”，而是“基于骨架图拓扑异常检测的笔画路径修正机制”。

### 4. 可写成论文贡献的表述

本文主要贡献可表述为：

1. 构建了一种面向书法机器人书写任务的轻量化字形图像到轨迹生成流程，实现从字形图像到机器人末端轨迹的自动转换。
2. 提出一种骨架拓扑异常感知的笔画轨迹修正方法，通过回绕率、方向连续性、端点可信度和部件边界约束，改善骨架路径中的误分割、误连接和交叉点路径选择不稳定问题。
3. 设计面向机器人执行的轨迹平滑与弧长重采样策略，在保持字形结构的同时提高轨迹连续性和可执行性。

### 5. 实验设计建议

为支撑上述创新点，实验不必依赖强化学习，可重点做以下对比：

- DFS 骨架追踪 vs 本文拓扑修正方法
- 无笔画先验 vs 加入笔画数与部件边界约束
- 无异常检测 vs 加入回绕率和方向连续性检测
- 无平滑 vs B样条 / Savitzky-Golay 平滑与弧长重采样

建议指标：

- 笔画数准确率
- 轨迹与骨架的 Chamfer 距离
- 轨迹渲染图与目标二值图的 IoU
- 轨迹平滑度（平均曲率或曲率突变次数）
- 异常回绕数量
- 平均运行时间

### 6. 当前开发优先级

当前应优先稳定非 RL 基线：

1. 先修 `stroke.py` 中的骨架拓扑解析和笔画路径配对。
2. 以“川”作为无交叉简单字验收，要求稳定输出 3 笔且无回绕。
3. 再处理“中”“永”等交叉结构字，重点修正交叉点分支配对。
4. 最后处理“福”等多部件复杂字，验证部件边界约束。
5. RL 模块暂时保留为探索性模块，除非后续结果稳定，否则不作为论文主创新。
